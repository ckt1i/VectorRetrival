# 精化报告

**日期**：2026-04-10（v3: 基于初步实验全面修正）

---

## 起点

用户提供的创新点：
1. 基于 ConANN 的聚类探测剪枝 + RaBitQ 误差界用于多阶段剪枝和预取阈值
2. 数据布局调整：FlatBuffers、向量+原始数据共存、聚类量化代码+地址按块存储
3. io_uring 解耦 CPU 与 I/O，实现向量搜索与数据预取的并行处理

## 精化轨迹

### 第 1 步：识别统一的洞察

三个创新点不是独立的优化——它们是单一洞察的表现：

> **量化误差界创建置信谱，可驱动 I/O 调度决策。**

- 第 1 点（ConANN + RaBitQ 界）提供**决策机制**
- 第 2 点（数据布局）提供 I/O 的**目标**
- 第 3 点（io_uring 重叠）提供决策的**执行模型**

没有误差界，布局只是存储工程。没有布局，界无处作用。没有重叠，决策无法高效执行。

### 第 2 步：建立贡献层级

**精化前**（平列表）：
- ConANN 剪枝 ≈ 数据布局 ≈ io_uring 重叠

**精化后**（层级）：
- **主导**：误差界驱动的三类 I/O 调度（界 → SafeOut/SafeIn/Uncertain 分类 → 差异化 I/O）
- **支撑**：协同设计存储布局（使 I/O 调度高效执行）
- **支撑**：io_uring 单线程重叠调度（使分类决策与 I/O 并行）
- **支撑**：多阶段量化 FastScan→ExRaBitQ（紧缩误差界，减少 Uncertain 率）

### 第 3 步：精化问题陈述

**前**："磁盘ANNS系统优化"
**后**："最小化磁盘向量搜索中的浪费 I/O，同时进行原始数据协同检索，利用量化误差界作为调度信号"

### 第 4 步：命名系统

**BoundFetch**——捕捉核心机制（界驱动取数决策）。

### 第 5 步：相对相关工作定位

| 系统 | 索引 | I/O 模型 | 原始数据处理 | 我们的优势 |
|------|------|---------|-----------|---------|
| DiskANN | 图（Vamana） | 扇区对齐读 | 搜索后获取 | 误差界驱动预取消除顺序原始数据往返 |
| SPANN | IVF+投票列表 | 批投票列表读 | 分离存储 | 分类减少浪费 I/O |
| Starling | 两层图 | 块读 | 未处理 | 原始数据协同检索作为一等公民 |
| PipeANN | 图 | SSD 对齐流水线 | 未处理 | IVF 批扫描比图遍历更易调度 |
| RaBitQ | （量化方法） | N/A | N/A | 将界从距离估算扩展到 I/O 调度 |

### 第 6 步：舍弃复杂性

舍弃的添加：
- ~~图索引~~ 指针追踪破坏批量 I/O 调度
- ~~多线程并发~~ 聚焦单查询延迟，吞吐量通过查询级并行正交扩展
- ~~学习路由~~ 误差界可证明，学习路由不可
- ~~GPU 卸载~~ I/O 是瓶颈，不是计算
- ~~过滤谓词~~ 正交关注，留作未来工作
- ~~多节点分布~~ 单节点范围足以证明贡献

### 第 7 步：基于初步实验的修正（v2→v3）

**初步实验揭示的关键事实**：

1. **SafeOut 是绝对主力**：Stage 1 即剪枝 90-98% 候选（随 top_k 变化）
2. **SafeIn 率当前 ≈ 0%**：但保留为三类框架的理论完备性和探索性贡献
3. **CRC 早停与 top_k 呈反向关系**：top_k=10 时早停 73%，top_k=20 时仅 21%
4. **单线程模型**：需明确作为设计范围写入文档

**对叙事的影响**：
- 原叙事："三类分类各司其职" → 修正："SafeOut 剪枝是核心价值，SafeIn 是潜力待挖掘的辅助机制"
- 增加 top_k 敏感性分析作为新的实验维度
- SafeIn 作为 C7 探索性声称保留

## 最终方法总结

BoundFetch 是一个磁盘 ANNS 系统，利用 RaBitQ 量化误差界将候选分三类 I/O 等级（SafeOut：跳过，Uncertain：仅向量+按需 payload，SafeIn：完整记录预取），通过基于 io_uring 的单线程重叠调度在协同设计的双文件存储布局上执行。初步实验表明 SafeOut 剪枝（90-98%）是 I/O 节省的主要来源，CRC 动态阈值在 top_k 较小时提供强力聚类早停（73%），SafeIn 的激活条件需进一步实验探索。
